# Sensor Dataset Validator

This project provides a Python pipeline for validating datasets generated by sensor-based data collection systems.

The script scans directories containing CSV files, extracts metadata, validates the quantity of files and rows for each sensor type, and detects signal interruptions.

It was designed as a reusable tool for validating experimental or wearable device datasets.

---

# Features

- Recursive dataset discovery
- Automatic metadata extraction
- Validation of expected sensor files
- Validation of minimum number of samples
- Detection of signal interruptions
- Generation of structured metadata tables

---

# Supported Sensors

The system can validate datasets containing sensors such as:

| Sensor | Expected Files | Minimum Samples |
|------|------|------|
| Accelerometer | 1 | 42000 |
| PPG | 1 | 85000 |
| BIA | 2 | 150 |
| ECG | 2 | 1500 |

These values can be modified in the configuration dictionaries:

```python
EXPECTED_FILES
EXPECTED_LINES
```

# Usage

Place your dataset inside a folder, for example:

```
data/
   subject01/
       accelerometer.csv
       ecg1.csv
       ecg2.csv
       ppg.csv
```

Run the script:

```bash
python sensor_dataset_validator.py
```
